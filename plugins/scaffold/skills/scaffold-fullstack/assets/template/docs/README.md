# docs/ — 設計方向文件

這個資料夾存放專案的**設計方向**:每個環境、功能或概念「為什麼這樣設計、有哪些取捨與邊界」。
它不是 API 參考,也不是教學;它是讓新人與 AI agent 快速理解專案走向、避免踩坑的地方。

## 工作慣例(請遵守)

> **每當你建立或改變一個環境、功能、或概念,就在這裡留下一份對應的設計方向文件。**

- 檔名用主題:`docs/<topic>.md`(例:`testing.md`、`import.md`、`auth.md`)。
- 內容重點是「why + 高層 how」;逐行細節留給程式碼註解。
- 寫完在下面的索引加一行。

詳見根目錄 `AGENTS.md` 的「邊建邊留設計方向文件」。

## 新人閱讀順序

1. 本檔(你在這)
2. [`architecture.md`](./architecture.md) —— 系統全景、分層、依賴規則、前後端牆
3. [`testing.md`](./testing.md) —— 測試策略與為什麼
4. 你要動的領域對應的 `docs/<topic>.md`
5. 程式碼(從 `Note` 垂直切片開始看最快)
